# Router

Choose the highest applicable tier; uncertainty moves upward.

- **Read-only:** answer or inspect without implementation artifacts; subject risk alone does not escalate.
- **Mechanical:** provably non-behavioral changes without data, dependency, security, public-contract, or external-state impact. Inspect, edit, check, review the diff, report.
- **Standard:** every non-mechanical implementation. Inspect state; state a brief plan and acceptance; implement minimally with practical focused coverage; run strongest focused checks; review final diff; report evidence.
- **High-risk:** follow Standard. Security-sensitive, destructive, production-data, migration, public-contract, cross-system, or consequential external-state work requires `writing-plans` before implementation or consequential action and `verification-before-completion`. Add `requesting-code-review` for material code or contract risk and `using-git-worktrees` when isolation protects user work.

User silence never waives Standard. Files or tools alone never escalate. Native plans need no durable file. Load extras only by tier or matching metadata.

Reclassify when new scope or risk appears and before consequential external action. Reuse fresh evidence unless state changed or a claim remains unproved.

Preserve user work; require authority for destructive or external actions; claim completion from fresh evidence.
